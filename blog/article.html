كيف تفهم Async/Await في JavaScript بأسلوب بسيط جداً

كود بالعربي
مطوّر ويب
2 يناير 2025 8 دقائق 1.2K قراءة

واحدة من أكثر المواضيع اللي تخبّل المبتدئين في JavaScript هي الـ Async/Await. كثير ناس تقرأ عنها وتحفظها، لكن ما تفهم ليش نستخدمها. في هذا المقال راح نحلها من الصفر.

قبل ما تكمل
المفروض تكون فاهم أساسيات JavaScript (functions، variables). لو ما وصلتها بعد، ارجع لمقال مقدمة في JavaScript أول.

المشكلة اللي حلّها Async/Await

تخيّل إنك تطلب بيانات من API — مثلاً قائمة المستخدمين. هذا الطلب يأخذ وقت. المشكلة إن JavaScript بطبيعتها لا تنتظر، تكمل تنفيذ الكود حتى لو البيانات ما وصلت بعد.

ما هو Async/Await بالضبط؟

بببساطة شديدة: async/await هو سكر نحوي فوق Promises — يعني تحته نفس الشيء، بس الكود أجمل وأسهل للفهم.

JavaScript 
// الطريقة القديمة بـ Promises
function getUser() {
  return fetch('/api/user')
    .then(res => res.json())
    .then(data => console.log(data))
    .catch(err => console.error(err));
}

// الطريقة الجديدة بـ Async/Await ✨
async function getUser() {
  try {
    const res  = await fetch('/api/user');
    const data = await res.json();
    console.log(data);
  } catch (err) {
    console.error(err);
  }
}

القواعد الأساسية (3 فقط)

  • كلمة async تُكتب قبل الـ function — وهذا يخلّي الدالة تُرجع Promise تلقائياً.
  • كلمة await تُستخدم داخل async فقط — وتخلّي JavaScript "تنتظر" النتيجة.
  • دائماً استخدم try/catch لمعالجة الأخطاء.
خطأ شائع
كثير مبتدئين يكتبون await خارج دالة async، هذا يعطيك error مباشرة!

مثال عملي من الواقع

JavaScript — مثال كامل 
async function getUsers() {
  try {
    const response = await fetch(
      'https://jsonplaceholder.typicode.com/users'
    );
    const users = await response.json();
    users.forEach((user) => {
      console.log(`👤 ${user.name}`);
    });
  } catch (error) {
    console.error('خطأ:', error);
  }
}
getUsers();

الخلاصة

  • async قبل الدالة → الدالة ترجع Promise
  • await قبل العملية → انتظر النتيجة
  • try/catch → لمعالجة الأخطاء
المقال القادم
سنتعلم كيف نستخدم Promise.all() مع Async/Await لتشغيل عدة طلبات بالتوازي. اشترك في النشرة ما تفوته!
